Student Performance Monitoring Pipeline

PythonSQLBigQueryGoogle Workspace APIsSeleniumGitHub ActionsPower BIStar SchemaMedallion Architecture

Problem

Academic performance data for 120+ schools across the district lived in scattered Excel sheets, updated inconsistently, with no single source of truth the District Collector or CEO could trust for decisions.

Architecture

  • Selenium scraper pulls student records from the EMIS government portal, the source data that seeds all 120 school tabs.
  • Python extracts HM-filled marks via Google Workspace APIs (Sheets, Drive, Gmail) into a Bronze layer, using MD5 row hashing to detect only real changes.
  • SQL validation layer (Silver) quarantines invalid records with structured reject codes, instead of failing the whole load.
  • Star schema Gold layer (1M+ rows) maintained via MERGE-based upserts — no full reloads.
  • 17 pre-computed analytics tables: district KPIs, block ranking, school performance categories (Good / Satisfactory / NI / Poor), subject failure combinations, attendance–pass correlation, and student risk flagging.
  • Automated HM notifications via Gmail API — each school receives a personalised email with a direct link to their own protected tab.
  • Orchestrated via GitHub Actions on a weekly schedule, with exponential backoff for API rate limits.

Impact

Replaced manual Excel-based reporting across 120 schools with an automated, analytics-ready system — giving district leadership a live, queryable view of students for the first time, turning a reporting lag of weeks into an on-demand view.

District School Performance Analysis

Cohort AnalysisStatistical SegmentationRoot-Cause AnalysisDistrict Reporting

Problem

Tracking academic performance from Class XI to Class XII across the district had no structured lens — no visibility into which schools or students were most likely to see performance drop between the two years, and no way to flag risk before results were out.

Methodology

  • Cohort transition analysis: paired every govt school student's Class XI Annual result with their Class XII Quarterly result to compute a school-level transition delta, isolating genuine decline from normal year-to-year variance.
  • Weighted contribution decomposition: broke the district pass percentage down by subject-group combination, quantifying each group's weighted pull on the aggregate — isolating that 3 of 14 groups accounted for a disproportionate share of the downside.
  • Outlier segmentation: classified schools into Critical (<50%), Weak (50–60%), Improving, and Declining bands based on transition delta, with drill-down to school × group granularity.
  • Root-cause targeting: cross-referenced the weakest subject-groups against attendance and pass-rate correlation to narrow interventions to the highest-leverage schools and groups.
  • Intervention design: translated findings into a Teach–Study–Test instructional cycle and a four-tier monitoring cascade (classroom → school → block → district), anchored to two KPIs — implementation rate and formative test score delta.
  • Findings and the monitoring framework were presented to the Chief Educational Officer, who directed implementation at the flagged schools.

Impact

12th standard govt school pass percentage rose from 89.71% to 93.40% in the following cycle, and the district's state rank climbed from 31st to 18th. The analysis directly informed the Chief Educational Officer's decision to target monitoring at the flagged schools and subject-groups, rather than apply a district-wide intervention.

Road Accident Intelligence Analytics

PythonGeocodingHotspot ClusteringPower BI

Problem

Road accident records existed as raw case entries with no spatial or trend view — no way to see where accidents cluster, or which stretches carry the highest fatality risk.

Architecture

  • Geocoded 7 years of accident records to map every case to a precise location.
  • Built hotspot clustering to surface high-frequency accident zones across the district.
  • Layered in fatality-rate analysis to separate high-volume spots from high-severity ones.
  • Power BI dashboard delivered to district administration for road safety planning.

Impact

Used in road safety planning decisions — turning raw case files into a spatial intelligence tool.
